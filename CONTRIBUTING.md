# Contribuindo para o projeto

## Como posso contribuir?

### O que aceitamos

É bem simples. Tente pensar em algo legal que gostaria de adicionar, ou algum bug que encontrou e deseja resolvê-lo. 
Aceitamos melhorias no código e documentação, boas práticas de programação e bug fixes.

### O que não será levado em consideração

PRs que não adicionam nada de importante ou interessante; sem avançar o estado atual do código ou projeto, ou que não apresentem uma explicação, motivo específico.
Se a alteração não segue o padrão estrututal do código, boas práticas e convenções já estabelecidas; será desconsiderada da mesma forma. 

## Pull Requests

1) Faça um fork do repositório e crie uma branch para implementar suas alterações;
2) Todo PR deve ter como alvo a branch ``develop``. Qualquer PR que não seguir isso, será fechado;
3) Toda feature ou bugfix deve vir seguida de testes;
4) Caso estiver resolvendo um issue, inclua isso no seu PR (exemplo: ``closes #45``);
5) Antes de subir qualquer alteração, leia atentamente aquilo que você fez :)

## Convenção de nomes para branches

- ``feat/adicionando-nova-feature``;
- ``refactor/aprimorando-estrutura-do-service-layer``;
- ``fix/fale-sobre-o-bugfix``;
- ``docs/melhorando-a-documentacao``

## Ficou alguma dúvida?

Abra uma discussão ou uma issue que poderemos te ajudar com quaisquer eventuais dúvidas.
